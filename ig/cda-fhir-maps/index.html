Introduction

This implementation guide provides maps to transform documents from CDA to FHIR and back using the FHIR Mapping Language (FML).
The maps are based on each other, first the transformation of the data types (FHIR Logical model CDA) is defined. Then based on this the mapping between the documents (CDA/FHIR), which becomes more and more specific (Swiss EPR Context, eMedication).

To execute the transformation of the documents locally, see the description here.
